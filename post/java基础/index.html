java比较杂的基础

一、jdk，JRE，JVM

以下为官方提供的java结构图

  • JDK(Java Development Kit，java开发工具包)

JDK包含了java 的所有内容，包括JRE和JVM。 比JRE包含了更多的是开发需要的工具，比如javac，javadoc等。

  • JRE(Java Runtime Environment, java运行环境)

java的所有程序需要在JRE（运行环境下）才能够运行。

包括JVM和java核心类库和支持文件

JRE比JDK少了那些开发的工具（如果不需要开发而只需要运行java 程序则有JRE就足够）

  • JVM（Java Virtual Machine, Java虚拟机）

JDK和JRE都包含着JVM。 因为java编译后的字节码是通过JVM和运行环境的操作系统做互动的，java之所以可以跨平台运行，都是因为JVM在操作系统和java程序之间进行映射

二、JavaSE,JavaEE,JavaME

  • JavaSE

标准版,主要用于开发和部署桌面、服务器以及嵌入式设备和实时环境中的Java应用程序。

JavaSE是JavaEE和JavaME的基础，用来开发C/S架构软件，通俗来讲，即开发电脑桌面应用软件，例如Eclipse

  • JavaEE

企业版，主要针对企业应用的开发,用来开发B/S架构软件。例如，电子商务网站

JavaEE在JavaSE的基础进行了扩展，增加了一些更加便捷的应用框架。比如我们现在常用的Java开发三大框架Spring、Struts和Hibernate，我们可以应用这些框架轻松写出企业级的应用软件。

  • JavaME

微型版，是一套运行专门为嵌入式设备设计的api接口规范，主要用于开发移动设备软件和嵌入式设备软件比如数字机顶盒，汽车导航系统
