Github项目搜索技巧

in

关键词in是用来限定搜索范围：

  • in:name: 指定搜索范围是仓库名称
  • in:description: 指定搜索范围是项目描述
  • in:readme: 指定搜索范围是readme文档

例子：

# 指定仓库名有sprintboot、mybatis
in:name springboot mybatis

stars、forks

通常判断一个项目好不好，是由stars和fork来判断

搜索方式如下：

  • stars(fork):> : 筛选stars(fork)数量大于某个值的仓库
  • stars(fork):start..end : 筛选stars(fork)数量在start和end之间的仓库

例子：

# 筛选描述有springboot，并且star大于100
in:description springboot stars:>100

language

指定项目的编写语言，如java、python

in:description 登录 language:java

created、pushed

in:name ssm pushed:>2019-12-02
